# GPFS (General Parallel File System)

## 基本信息
- 发布时间：1998 年（IBM Research）
- 开发方：IBM
- 开源/商用：商用（后以 Spectrum Scale 品牌发布，部分组件开源）
- 核心论文：*Design and Implementation of GPFS* (2002)
- 当前版本：IBM Spectrum Scale 5.x

## 架构设计

### 核心架构
```
┌─────────────────────────────────────────────┐
│              GPFS Cluster                    │
├─────────────────────────────────────────────┤
│  NSD (Network Shared Disk)                  │
│  ┌────────┐  ┌────────┐  ┌────────┐        │
│  │ Disk 1 │  │ Disk 2 │  │ Disk N │        │
│  └────────┘  └────────┘  └────────┘        │
│       ↑           ↑           ↑              │
│  ┌────────┐  ┌────────┐  ┌────────┐        │
│  │ NSD    │  │ NSD    │  │ NSD    │        │
│  │ Server │  │ Server │  │ Server │        │
│  └────────┘  └────────┘  └────────┘        │
│       ↓           ↓           ↓              │
│  ┌────────────────────────────────────┐     │
│  │      GPFS Client Nodes              │     │
│  │  (Parallel Access to NSDs)          │     │
│  └────────────────────────────────────┘     │
└─────────────────────────────────────────────┘
```

### 节点角色
- **NSD Server**: 管理物理磁盘，为客户端提供块级访问
- **Client Node**: 挂载 GPFS 文件系统，并行访问数据
- **Manager Node**: 集群管理节点（主/备）
- **Token Manager**: 管理分布式锁

## 核心技术

### 1. 并行 I/O 架构
- 所有客户端节点可**同时**读写同一文件系统
- 数据在多个磁盘间条带化（striping），类似 RAID-0
- I/O 路径绕过 NSD Server 直接访问磁盘（direct I/O）

### 2. 分布式锁管理 (DLM)
- 基于 **token** 的分布式锁机制
- Token 类型：读锁、写锁
- 节点获取对应 token 后才能操作文件/数据块
- 锁粒度：文件级、字节范围级

### 3. 条带化 (Striping)
- 文件数据块分布到多个 NSD
- 可配置条带宽度（block size）和条带数量
- 大文件自动并行化，提升吞吐

## 元数据架构：分布式条带化

**模式归类**：分布式元数据（非集中式、非无中心）

- **分布方式**：元数据和数据同样条带化分布在所有 NSD 上
- **inode**：标准 inode 结构，扩展支持分布式
- **目录结构**：目录项也分布存储，非集中式 MDS
- **日志**：每个节点维护本地日志，元数据更新原子性
- **锁机制**：Token-based DLM 协调并发元数据操作
- **特点**：元数据不成为瓶颈（区别于 Lustre 集中式 MDS）

## 数据架构：共享存储 + 条带化

**模式归类**：共享存储（NSD）+ 本地分布式

- **存储介质**：NSD（Network Shared Disk），可为本地磁盘或 SAN
- **条带化**：数据块分布到多个 NSD，可配置条带宽度
- **Direct I/O**：I/O 路径绕过 NSD Server 直接访问磁盘
- **并行 I/O**：所有客户端同时读写，类似 RAID-0 跨网络

## 一致性模型

- **强一致性**：POSIX 语义
- Token 机制确保并发正确性
- 读写一致性通过锁协议保证

## 容错与高可用

- **NSD 副本**：数据可在多个 NSD 上镜像
- **Manager 高可用**：主备切换
- **磁盘故障**：NSD 级别的故障转移
- 无内置纠删码（后来版本添加了 RAID 支持）

## 性能特征

| 指标 | 典型值 |
|------|--------|
| 单文件系统规模 | PB 级 |
| 最大节点数 | 数千 |
| 最大文件数 | 数十亿 |
| 吞吐量 | GB/s 级（聚合） |
| 延迟 | ms 级 |
| 元数据架构 | 分布式条带化 |
| 数据架构 | 共享存储 (NSD) |

## 适用场景

- **最佳**：HPC 高性能计算、大规模科学计算
- **局限**：部署复杂、商用许可成本高、对小文件优化不足

## 历史影响

1. **并行 I/O 范式**：开创了并行文件系统先河
2. **metadata/data 分离思想**：影响后续 Lustre 等系统设计
3. **条带化分布**：被 Ceph、HDFS 等系统以不同形式借鉴
4. **token-based 锁**：影响后续分布式锁设计

## 局限与教训

- 商用闭环限制开源生态发展
- 部署运维复杂度高
- 面向 HPC 设计，对云原生/弹性场景适配不足
- 小文件性能不够理想
